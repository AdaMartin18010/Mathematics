## 概念定义 | Concept Definition

### 可计算性分层 | Computability Hierarchy

- 中文定义：可计算性分层研究问题能否被算法解决，复杂性分层则进一步区分可计算问题的资源消耗（如时间、空间）。复杂性理论通过分层（如P、NP、PSPACE等）刻画问题的难度，是理论计算机科学的核心内容。
- English Definition: Computability hierarchy studies whether problems can be solved by algorithms, while complexity hierarchy further distinguishes the resource consumption (such as time and space) of computable problems. Complexity theory characterizes the difficulty of problems through hierarchies (such as P, NP, PSPACE, etc.), forming a core part of theoretical computer science.
- 国际标准/权威来源：
  - ISO/IEC 2382:2015 (Information technology — Vocabulary)
  - Stanford Encyclopedia of Philosophy: Complexity Theory, Computability
  - Encyclopedia of Mathematics: Complexity class, Computability
  - Wikipedia: Computational complexity theory, Complexity class
- 相关批判与哲学反思：
  - P=NP问题是复杂性理论的核心难题，影响密码学、算法设计等众多领域
  - 复杂性分层揭示了理论与实际计算之间的鸿沟
  - 资源受限模型（如空间、时间）反映了现实计算的约束
  - 复杂性理论的实际应用与理论边界仍在不断探索中

### 复杂性类 | Complexity Classes

- 中文定义：复杂性类是计算复杂性理论中的基本概念，用于分类计算问题根据其解决所需的资源（如时间、空间）的多少。
- English Definition: Complexity classes are fundamental concepts in computational complexity theory, used to classify computational problems according to the amount of resources (such as time and space) required to solve them.
- 国际标准/权威来源：
  - ISO/IEC 2382:2015 (Information technology — Vocabulary)
  - Stanford Encyclopedia of Philosophy: Complexity Classes
  - Encyclopedia of Mathematics: Complexity Classes
  - Wikipedia: Complexity class
- 相关批判与哲学反思：
  - 复杂性类虽然提供了分类框架，但可能简化了实际问题的复杂性
  - 理论分类与实际应用之间存在张力

### P类问题 | P Class Problems

- 中文定义：P类问题是指可以在多项式时间内被确定性图灵机解决的问题，是复杂性理论中最基本的复杂性类之一。
- English Definition: P class problems are those that can be solved by deterministic Turing machines in polynomial time, one of the most fundamental complexity classes in complexity theory.
- 国际标准/权威来源：
  - ISO/IEC 2382:2015 (Information technology — Vocabulary)
  - Stanford Encyclopedia of Philosophy: P Class
  - Encyclopedia of Mathematics: P Class
  - Wikipedia: P (complexity)
- 相关批判与哲学反思：
  - P类问题虽然理论上可解，但实际中可能仍然困难
  - 多项式时间与实际问题规模之间存在张力

### NP类问题 | NP Class Problems

- 中文定义：NP类问题是指可以在多项式时间内被非确定性图灵机解决的问题，或者其解可以在多项式时间内被验证的问题。
- English Definition: NP class problems are those that can be solved by non-deterministic Turing machines in polynomial time, or whose solutions can be verified in polynomial time.
- 国际标准/权威来源：
  - ISO/IEC 2382:2015 (Information technology — Vocabulary)
  - Stanford Encyclopedia of Philosophy: NP Class
  - Encyclopedia of Mathematics: NP Class
  - Wikipedia: NP (complexity)
- 相关批判与哲学反思：
  - NP类问题虽然理论上可解，但实际中可能非常困难
  - 非确定性与实际计算之间存在张力

### NP完全问题 | NP-Complete Problems

- 中文定义：NP完全问题是NP类中最困难的问题，任何NP问题都可以在多项式时间内归约到NP完全问题。
- English Definition: NP-complete problems are the hardest problems in the NP class, to which any NP problem can be reduced in polynomial time.
- 国际标准/权威来源：
  - ISO/IEC 2382:2015 (Information technology — Vocabulary)
  - Stanford Encyclopedia of Philosophy: NP-Complete
  - Encyclopedia of Mathematics: NP-Complete
  - Wikipedia: NP-completeness
- 相关批判与哲学反思：
  - NP完全问题虽然理论上有解，但实际中可能无法有效解决
  - 理论可解性与实际可行性之间存在张力

### 递归函数 | Recursive Functions

- 中文定义：递归函数是指可以通过基本函数和递归规则构造的函数，是可计算性理论的核心概念。
- English Definition: Recursive functions are functions that can be constructed from basic functions and recursive rules, a core concept in computability theory.
- 国际标准/权威来源：
  - ISO/IEC 2382:2015 (Information technology — Vocabulary)
  - Stanford Encyclopedia of Philosophy: Recursive Functions
  - Encyclopedia of Mathematics: Recursive Functions
  - Wikipedia: Recursive function
- 相关批判与哲学反思：
  - 递归函数虽然提供了计算模型，但可能简化了实际计算的复杂性
  - 递归与迭代之间存在张力

### 5.2 库克与NP完全性 | Cook & NP-Completeness

**代表人物与贡献：**

- 斯蒂芬·库克（Stephen Cook, 1939-）
- 发展了NP完全性理论
- 提出了库克定理

**原话引用：**
> "The P versus NP problem is the most important open problem in computer science."
> "P与NP问题是计算机科学中最重要的开放问题。" — 库克

**NP完全性框架：**

```lean
-- P类问题的定义
def P_class : Problem → Prop
| problem => exists (algorithm : Algorithm),
    polynomial_time algorithm ∧ solves algorithm problem

-- NP类问题的定义
def NP_class : Problem → Prop
| problem => exists (verifier : Algorithm),
    polynomial_time verifier ∧ verifies verifier problem

-- NP完全性定义
def NP_complete : Problem → Prop
| problem => NP_class problem ∧ 
    forall (other_problem : Problem), 
      NP_class other_problem → reduces_to other_problem problem
```

### 5.3 卡普与NP完全性 | Karp & NP-Completeness

**代表人物与贡献：**

- 理查德·卡普（Richard Karp, 1935-）
- 发展了NP完全性理论
- 提出了多个NP完全问题

**原话引用：**
> "The theory of NP-completeness provides a powerful tool for showing that problems are computationally intractable."
> "NP完全性理论为证明问题的计算不可行性提供了强大的工具。" — 卡普

**NP完全性证明框架：**

```lean
-- 多项式时间归约
def polynomial_reduction : Problem → Problem → Prop
| P1, P2 => exists (reduction : Algorithm),
    polynomial_time reduction ∧ 
    forall (input : string),
      P1 input ↔ P2 (reduction input)

-- NP完全性证明
theorem NP_completeness_proof : Problem → Prop
| problem => NP_class problem ∧ 
    forall (other_problem : Problem),
      NP_class other_problem → polynomial_reduction other_problem problem
```

### 5.4 萨维奇与PSPACE | Savitch & PSPACE

**代表人物与贡献：**

- 沃尔特·萨维奇（Walter Savitch, 1943-）
- 发展了PSPACE理论
- 提出了萨维奇定理

**原话引用：**
> "Space complexity provides a different perspective on computational difficulty."
> "空间复杂性为计算困难性提供了不同的视角。" — 萨维奇

**PSPACE框架：**

```lean
-- PSPACE类问题的定义
def PSPACE_class : Problem → Prop
| problem => exists (algorithm : Algorithm),
    polynomial_space algorithm ∧ solves algorithm problem

-- 萨维奇定理
theorem savitch_theorem : Prop :=
  forall (problem : Problem),
    NPSPACE_class problem → PSPACE_class problem
```

### 5.5 西普瑟与复杂性理论 | Sipser & Complexity Theory

**代表人物与贡献：**

- 迈克尔·西普瑟（Michael Sipser, 1954-）
- 发展了复杂性理论
- 编写了经典教材

**原话引用：**
> "Complexity theory is the study of the intrinsic difficulty of computational problems."
> "复杂性理论是研究计算问题内在难度的学科。" — 西普瑟

**复杂性理论框架：**

```lean
-- 复杂性类层次结构
def complexity_hierarchy : ComplexityClass → ComplexityClass → Prop
| C1, C2 => forall (problem : Problem),
    C1 problem → C2 problem

-- 复杂性分离
def complexity_separation : ComplexityClass → ComplexityClass → Prop
| C1, C2 => exists (problem : Problem),
    C2 problem ∧ not (C1 problem)
```
