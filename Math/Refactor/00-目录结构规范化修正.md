# 目录结构规范化修正方案

## 当前问题分析

### 1. 目录结构问题

- **重复目录**：存在多个相同主题的目录（如 `01-数学哲学/` 和 `01-数学哲学基础.md`）
- **序号混乱**：目录序号不连续，存在 `01-`、`02-`、`03-`、`04-`、`05-`、`07-`、`08-` 等
- **文件位置不当**：部分文件直接放在主题目录下，应该放在子目录中
- **命名不一致**：文件名格式不统一

### 2. 内容结构问题

- **重复内容**：多个文件包含相同或相似的内容
- **引用不规范**：缺少统一的交叉引用和本地跳转
- **目录不完整**：部分文件缺少严格的树形序号目录

### 2. 文件命名规范

#### 2.1 目录命名

- 格式：`NN-主题名称/`
- 示例：`01-哲学与元数学基础/`

#### 2.2 文件命名

- 总览文件：`00-主题总览.md`
- 子主题文件：`NN-具体内容.md`
- 示例：`01-数学哲学.md`

### 3. 内容结构规范

#### 3.1 文件头部结构

```markdown
# 标题

## 目录

- [标题](#标题)
  - [目录](#目录)
  - [1. 引言](#1-引言)
  - [2. 主要内容](#2-主要内容)
    - [2.1 子内容1](#21-子内容1)
    - [2.2 子内容2](#22-子内容2)
  - [3. 总结](#3-总结)
  - [4. 参考文献](#4-参考文献)

## 1. 引言

## 2. 主要内容

### 2.1 子内容1

### 2.2 子内容2

## 3. 总结

## 4. 参考文献
```

#### 3.2 交叉引用规范

- 本地跳转：`[见 2.1.3](../02-数学基础与逻辑/01-集合论/02-基数理论.md#213-基数比较)`
- 锚点格式：`#NNN-标题内容`

### 4. 修正执行计划

#### 4.1 第一阶段：目录结构修正

1. 创建标准化的目录结构
2. 移动和重命名文件
3. 删除重复目录和文件

#### 4.2 第二阶段：内容规范化

1. 统一文件格式和结构
2. 建立交叉引用系统
3. 完善目录和索引

#### 4.3 第三阶段：质量检查

1. 检查内容一致性
2. 验证交叉引用
3. 完善文档结构
