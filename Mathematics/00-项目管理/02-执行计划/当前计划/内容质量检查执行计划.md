## 📈 检查维度

### 1. 内容准确性（40%权重）

**检查项**:

- [ ] 数学定义是否严格准确？
- [ ] 定理是否有完整证明？
- [ ] 概念解释是否清晰？
- [ ] 示例是否正确？
- [ ] 术语使用是否一致？

**评估标准**:

- A级: 所有定义严格准确，所有定理有证明
- B级: 核心定义准确，主要定理有证明
- C级: 定义基本正确，部分定理缺乏证明

### 2. 形式化程度（20%权重）

**检查项**:

- [ ] 是否有形式化定义？
- [ ] 是否有形式化证明？
- [ ] 是否与Lean mathlib对齐？
- [ ] 形式化覆盖率如何？

**评估标准**:

- A级: 形式化覆盖率>80%
- B级: 形式化覆盖率>50%
- C级: 形式化覆盖率>5%

### 3. 结构组织（15%权重）

**检查项**:

- [ ] 目录结构是否清晰？
- [ ] 导航是否顺畅？
- [ ] 文档组织是否合理？
- [ ] 交叉引用是否准确？

### 4. 用户体验（15%权重）

**检查项**:

- [ ] 内容是否易懂？
- [ ] 学习路径是否清晰？
- [ ] 示例是否充足？
- [ ] 交互是否友好？

### 5. 文档完整性（10%权重）

**检查项**:

- [ ] 文档结构是否完整？
- [ ] 内容是否完整？
- [ ] 链接是否有效？
- [ ] 格式是否规范？

## 📝 问题分类

### P0问题（紧急）

- 数学定义错误
- 定理证明错误
- 严重格式问题
- 链接完全失效

### P1问题（重要）

- 定义不够严格
- 证明不完整
- 术语不一致
- 格式不规范

### P2问题（一般）

- 示例不够典型
- 解释不够清晰
- 结构可以优化
- 链接可以改进
