## 📋 检查清单概述

本检查清单用于确保项目内容符合质量标准，分为每日、每周、每月三个级别。

---

## 📅 每日检查清单

### 内容质量检查

- [ ] **数学定义准确性**
  - [ ] 所有数学定义严格准确
  - [ ] 术语使用一致
  - [ ] 符号使用规范

- [ ] **证明完整性**
  - [ ] 主要定理有完整证明
  - [ ] 证明过程逻辑清晰
  - [ ] 无sorry占位符（Lean代码）

- [ ] **文档格式**
  - [ ] Markdown格式正确
  - [ ] 数学公式渲染正常
  - [ ] 链接有效性

### 进度跟踪

- [ ] 更新任务进度
- [ ] 记录遇到的问题
- [ ] 保存工作状态
- [ ] 更新状态跟踪文档

---

## 📆 每周检查清单

### 内容质量评估

- [ ] **模块完成度评估**
  - [ ] 评估各模块真实完成度
  - [ ] 识别缺失内容
  - [ ] 制定补充计划

- [ ] **质量评级更新**
  - [ ] 评估当前质量等级
  - [ ] 识别改进方向
  - [ ] 制定提升计划

- [ ] **链接验证**
  - [ ] 检查所有内部链接
  - [ ] 修复断链
  - [ ] 更新过时链接

### 进度总结

- [ ] 总结本周成果
- [ ] 评估目标达成度
- [ ] 规划下周任务
- [ ] 更新路线图
- [ ] 更新进度报告

---

## 📊 每月检查清单

## 📈 质量指标检查

### 内容准确性指标

- [ ] 数学定义准确率 ≥ 90%
- [ ] 定理证明完整率 ≥ 95%
- [ ] 示例正确率 = 100%
- [ ] 术语一致性 ≥ 95%

### 形式化程度指标

- [ ] 形式化覆盖率 ≥ 50%（6个月目标）
- [ ] Lean代码完整性 = 100%
- [ ] mathlib对齐度 ≥ 80%

### 结构组织指标

- [ ] 目录结构清晰度 ≥ 90%
- [ ] 导航系统有效性 ≥ 90%
- [ ] 链接有效性 ≥ 98%
- [ ] 重复内容率 ≤ 5%
