## 📋 格式规范概述

本规范定义了Mathematics项目中所有文档的格式要求，确保文档格式统一、清晰、易读。

---

## 📝 文件命名规范

### 1. 命名原则

- ✅ 使用中文或英文名称
- ✅ 名称清晰描述内容
- ✅ 使用连字符分隔单词
- ✅ 避免特殊字符
- ✅ 使用数字前缀排序

### 2. 命名格式

**标准格式**:

```
[序号]-[描述性名称].md
```

**示例**:

- ✅ `01-项目路线图.md`
- ✅ `02-执行计划.md`
- ✅ `01-数学基础.md`
- ✅ `连续函数定义.md`

**不良示例**:

- ❌ `项目路线图2025.md` (缺少序号)
- ❌ `执行 计划.md` (有空格)
- ❌ `project_plan.md` (中英文混用不一致)

---

## 📄 文档头部规范

### 标准头部格式

每个文档必须包含以下头部信息：

```markdown
# 文档标题

**创建日期**: YYYY-MM-DD
**版本**: v1.0
**状态**: 草案/进行中/完成
**适用范围**: [说明]

---
```

### 头部字段说明

- **创建日期**: 文档首次创建的日期
- **版本**: 文档版本号（v1.0, v1.1等）
- **状态**: 文档当前状态
- **适用范围**: 文档适用的范围或模块

---

## 📐 标题层级规范

### 标题层级

使用标准的Markdown标题层级：

```markdown
# 一级标题（文档标题）

## 二级标题（主要章节）

### 三级标题（子章节）

#### 四级标题（小节）

##### 五级标题（子小节）
```

### 标题规范

- ✅ 一级标题只有一个（文档标题）
- ✅ 标题层级连续，不跳跃
- ✅ 标题描述清晰准确
- ✅ 同级标题格式一致

## 📋 列表规范

### 1. 无序列表

**标准格式**:

```markdown
- 项目1
- 项目2
- 项目3
```

**嵌套列表**:

```markdown
- 主项目1
  - 子项目1.1
  - 子项目1.2
- 主项目2
```

### 2. 有序列表

**标准格式**:

```markdown
1. 第一项
2. 第二项
3. 第三项
```

### 3. 任务列表

**标准格式**:

```markdown
- [ ] 未完成任务
- [x] 已完成任务
```

## 🔢 数学公式规范

### 1. 行内公式

**标准格式**:

```markdown
函数 $f(x) = x^2$ 是一个二次函数。
```

### 2. 块级公式

**标准格式**:

```markdown
$$
f(x) = \int_{-\infty}^{\infty} \hat{f}(\xi) e^{2\pi i \xi x} d\xi
$$
```

### 3. 编号公式

**标准格式**:

```markdown
$$
f(x) = x^2 + 2x + 1 \tag{1}
$$
```

## 📊 表格规范

### 标准表格格式

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据1 | 数据2 | 数据3 |
| 数据4 | 数据5 | 数据6 |
```

### 表格对齐

- `|:---|` 左对齐
- `|:---:|` 居中对齐
- `|---:|` 右对齐

## 🔗 链接规范

### 1. 内部链接

**标准格式**:

```markdown
[链接文本](./相对路径/文件.md)
[链接文本](./相对路径/文件.md#锚点)
```

### 2. 外部链接

**标准格式**:

```markdown
[链接文本](https://example.com)
[链接文本](https://example.com "链接说明")
```

## 💻 代码块规范

### 1. 行内代码

**标准格式**:

```markdown
使用 `function_name()` 函数。
```

### 2. 代码块

**标准格式**:

````markdown
```language
代码内容
```
````

**示例**:

````markdown
```lean
def add (a b : ℕ) : ℕ := a + b
```
````

## 🎨 强调规范

### 文本强调

- `*斜体*` 或 `_斜体_` → *斜体*
- `**粗体**` 或 `__粗体__` → **粗体**
- `***粗斜体***` → ***粗斜体***

### 使用场景

- **粗体**: 强调重要概念、关键词
- **斜体**: 强调术语、变量名
- **粗斜体**: 特殊强调

---
